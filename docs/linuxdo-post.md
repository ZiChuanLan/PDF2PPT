# PDF2PPT：把扫描版 PDF / 课件截图 / 单张图片，尽量转成可编辑 PPTX

项目地址：<https://github.com/ZiChuanLan/PDF2PPT>

这不是简单把 PDF 截成一张背景图塞进 PPT。

它更适合这些场景：

- 扫描版 PDF，想尽量转成还能继续改字的 PPT
- 课程讲义、汇报材料、截图课件，想快速整理成演示文稿
- 图片很多、版面比较固定的文档，想保留原来的视觉布局
- 想自己部署一个网页工具，不想手动跑一堆脚本

下面这篇只讲三件事：

- 这个项目能干嘛
- 页面长什么样
- 小白怎么最快跑起来

不讲底层原理，不讲 OCR 架构，不讲复杂实现。

---

## 一句话看懂

你可以把 `PDF` 或 `单张图片` 丢进去，项目会尽量帮你生成一个 `PPTX`，并且提供：

- 上传与预览
- 单页试跑
- OCR 识别配置
- 任务跟踪
- 过程图保留
- 结果下载

## 先看界面

说明一下：下面几张图主要是帮助理解页面布局和操作入口，自动截图环境里的字体渲染和你自己部署后的页面可能会有一点差别，但功能位置是一致的。

### 1. 首页：上传、预览、开始转换

![首页上传与预览](https://raw.githubusercontent.com/ZiChuanLan/PDF2PPT/main/docs/assets/linuxdo/home-upload-full.png)

这一页主要就是干活入口：

- 直接拖 PDF 或图片进去
- 可以先看预览
- PDF 可以整份处理，也可以单页试跑
- 图片会自动按单页处理
- 一键开始转换

我自己比较喜欢这个点：

- 不用先理解一堆参数
- 首页只保留最常用的东西
- 更细的配置放到设置页，不会一上来把人劝退

### 2. 设置页：常用参数都在这里

![设置页](https://raw.githubusercontent.com/ZiChuanLan/PDF2PPT/main/docs/assets/linuxdo/settings.png)

设置页主要解决的是“效果要不要更稳、更快、更可控”：

- 选择解析引擎
- 选择 OCR 方案
- 填 API Key / Base URL / 模型
- 调整图片处理方式
- 调整 PPT 生成模式
- 需要时再展开高级参数

适合两类人：

- 想直接用默认配置快速开跑的人
- 想自己调效果的人

### 3. 跟踪页：看进度、看结果、做排障

![跟踪页](https://raw.githubusercontent.com/ZiChuanLan/PDF2PPT/main/docs/assets/linuxdo/tracking.png)

这个页面是我觉得很实用的部分：

- 能看到任务列表
- 能看状态、阶段、进度
- 后面可以专门用来核对输出结果
- 开启过程图时，更适合排查“为什么这一页没识别好”

## 它现在能做什么

### 输入支持

- PDF
- PNG
- JPG / JPEG
- WEBP

其中图片上传的逻辑是：

- 直接上传单张图片
- 系统会自动按单页处理
- 然后继续走现有的转换流程

### 输出结果

- 生成 `PPTX`
- 可在网页中跟踪任务状态
- 可下载最终结果
- 需要时保留过程图和中间产物

### 适合的内容类型

- 扫描文档
- 课件截图
- 图片版讲义
- 图文混排材料
- 需要重新编辑的汇报底稿

### 页面层面的实用功能

- 首页上传
- 文档预览
- 单页试跑
- 设置页参数保存
- 任务进度跟踪
- 结果核对
- 调试过程图保留

## 小白怎么最快上手

下面这个流程就够了。

### 1. 拉代码

```bash
git clone https://github.com/ZiChuanLan/PDF2PPT.git
cd PDF2PPT
```

### 2. 准备环境变量

```bash
cp .env.example .env
```

默认情况下先不用把所有东西都填满。

### 3. 用 Docker 直接启动

```bash
docker compose up -d --build
```

### 4. 打开网页

```text
http://localhost:3000
```

如果你没有改 `.env`，默认网页访问密码是：

```text
123456
```

只是本地体验可以先这样，正式部署一定要改掉。

### 5. 开始第一次转换

建议第一次这样试：

1. 先上传一份页数少的 PDF，或者直接上传一张图片
2. 首页先别动太多高级选项
3. 先点单页试跑，看看效果
4. 如果方向对，再整份处理
5. 转完后去跟踪页看结果并下载 PPTX

## 新手推荐配置

如果你不想研究太久，建议先从这套组合开始：

- 解析引擎：`AIOCR`
- 页面图片处理方式：`图片拆出来（可单独编辑）`
- PPT 生成模式：`快速`

模型这块可以优先试：

- `deepseek-ai/deepseek-ocr`
- `PaddlePaddle/PaddleOCR-VL-1.5`

这套配置的思路很简单：

- 先保证能跑通
- 再追求效果
- 别一上来就把所有高级参数都调满

## 我觉得比较实用的几个点

### 1. 不只限 PDF

现在不仅能传 PDF，也能直接传图片。

对很多人来说其实更方便，因为很多真实材料根本不是规范 PDF，而是：

- 微信里存下来的课件图
- 截图版讲义
- 从别的平台导出的单页图片

### 2. 首页和设置页分工比较清楚

这点对非技术用户很重要：

- 首页负责“先把任务跑起来”
- 设置页负责“需要时再调”

不会一打开就被几十个参数吓退。

### 3. 有任务跟踪，不是盲等

很多工具最烦的就是：

- 点完按钮
- 不知道在干嘛
- 卡住了也不知道卡哪

这个项目至少给了一个明确的任务跟踪入口，排障会轻松很多。

### 4. 可以保留过程图

如果某一页效果不对，过程图非常有帮助。

比如你可以更快判断：

- 是 OCR 没识别出来
- 还是图片区域切分有问题
- 还是最后合成成 PPT 的时候出了偏差

## 现在更适合怎样使用

如果你问我这项目最适合什么使用方式，我会给一个很现实的答案：

- 更适合自部署
- 更适合自己或小团队内部用
- 更适合先拿真实材料小范围验证

它不是那种“什么文档都 100% 完美还原”的工具。

但如果你的目标是：

- 尽量省人工重做 PPT 的时间
- 尽量把能编辑的东西保留下来
- 有页面可用、有任务可追踪、有结果可下载

那它是很值得体验的一类项目。
